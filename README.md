# 🏦 Bank Management System (OOP)

مشروع تطبيق إدارة بنكي متكامل يعتمد على مفاهيم البرمجة كائنية التوجه (OOP) بلغة C++. يوفر النظام بيئة آمنة ومنظمة لإدارة حسابات العملاء وصلاحيات المستخدمين مع الاحتفاظ بسجلات كاملة للعمليات.

## ✨ الميزات الرئيسية (Features)

### 👥 إدارة العملاء (Client Management)
- إضافة، تحديث، وحذف بيانات العملاء.
- البحث عن عميل وعرض قائمة العملاء الكاملة.
- **العمليات المالية (Transactions):** السحب، الإيداع، التحويل بين الحسابات، وعرض إجمالي الأرصدة.

### 🔐 نظام المستخدمين والصلاحيات (User & Permissions System)
- شاشة تسجيل دخول آمنة تدعم عدد محاولات محدود (3 محاولات).
- إدارة كاملة للمستخدمين (إضافة، تعديل، حذف، بحث).
- نظام صلاحيات متقدم يمنع أو يسمح للمستخدم بدخول شاشات معينة بناءً على رتبته.
- **سجل العمليات (Login Register):** تتبع عمليات تسجيل الدخول الناجحة والفاشلة مع التوقيت وحساب المستخدم.

### نظام حماية لكلمات المرور الخاصة بكل من المستخدمين والعملاء.
### شاشة صيرفة خاصة تقوم بعرض جميع العملات الدولية مع مميزات مثل البحث والتعديل على الRate وكذلك لحساب ومقارنة العملات اعتمادا على الدولار.

## 🛠️ المفاهيم البرمجية المستخدمة (OOP Concepts Applied)
- **الوراثة (Inheritance):** استخدام كلاسات أساسية ومستمدة (مثل كلاس الشاشات الأساسي `clsScreen`).
- **التغليف (Encapsulation):** حماية البيانات داخل الكلاسات باستخدام الصلاحيات (`private`, `protected`, `public`).
- **التجريد (Abstraction):** فصل العمليات المعقدة لتبسيط واجهات التعامل.
- **فصل المهام:** فصل منطق واجهة المستخدم (UI) عن منطق معالجة البيانات (Business Logic).

---

## 📂 هيكلة ملفات المشروع (Project Structure)

- `main.cpp`: نقطة الانطلاق وتشغيل البرنامج.
- `clsScreen.h`: الكلاس الأساسي لجميع الشاشات لتنظيم العرض والعناوين.
- `clsMainScreen.h`: الشاشة الرئيسية التي تتفرع منها شاشات النظام.
- `clsBankClient.h` & `clsUser.h`: الكلاسات المسؤولة عن معالجة بيانات العملاء والمستخدمين وتخزينها في الملفات.
